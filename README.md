# 🪨 Token Tracker (Codex Dashboard)

> **Codex 전용 초경량 실시간 토큰 대시보드**  
> AI 코딩 중 소모되는 토큰량과 비용을 100% 무자각으로 감시하고 시각화합니다.
> 복잡한 프록시 설정이나 오류 없이 명령어 단 하나로 모든 환경을 제어합니다.

---

## 🚀 다운로드 및 3초 초간단 설치법 (Windows)

Token Tracker를 처음 사용하신다면 다음 순서대로 진행해 주세요.

### 1️⃣ 프로젝트 다운로드
파워쉘(PowerShell)이나 명령 프롬프트(CMD)를 열고 아래 명령어를 입력하여 프로젝트를 다운로드합니다.
```bash
git clone https://github.com/bucheoncityboy/token-tracker.git
cd token-tracker
```
*(Git이 설치되어 있지 않다면, GitHub 페이지 우측 상단의 `Code` 버튼을 누르고 `Download ZIP`을 통해 다운로드 후 압축을 해제하셔도 됩니다.)*

### 2️⃣ 원클릭 설치 스크립트 실행
프로젝트 폴더 내에 포함된 **`setup.bat`** 파일을 마우스로 **더블클릭**만 하시면 모든 설치가 끝납니다!

**자동으로 완료되는 작업 내용:**
1. 필요한 패키지 모듈 1초 컷 자동 설치 (`npm install`)
2. PowerShell 전역 **`token`** 단축 명령어 프로필 자동 등록
3. 터미널 한글 인코딩(UTF-8) 깨짐 완벽 방지 세팅 적용

> [!NOTE]
> 수동으로 설치하고 싶으시다면 프로젝트 폴더에서 파워쉘을 열고 다음 명령어를 실행해 주세요:
> ```powershell
> npm install; node src/setup-profile.js; Write-Host "`n✓ Installation complete!" -ForegroundColor Green
> ```

## ⚡ 핵심 사용법 & 워크플로우

설치가 끝난 후 파워쉘 터미널을 새롭게 열고 **`token`**을 입력하면 대시보드 메뉴가 즉시 열립니다.

```powershell
token
```

### 🔄 표준 워크플로우

```
[1] 대시보드 가동 ──▶ 💻 Codex로 평소처럼 코딩 ──▶ 📊 실시간 토큰 확인
```

1. **`[1]` 대시보드 가동 (Open Real-time Dashboard)**
   * 실시간 다크 테마 글래스모피즘 웹 대시보드 서버가 백그라운드에 구동되고 브라우저 창이 열립니다.
2. **`💻` 자유로운 코딩**
   * VS Code, Cursor 등 사용하시는 IDE에서 평소처럼 Codex로 코딩하세요. Token Tracker가 백그라운드에서 조용히 소모되는 토큰을 모니터링합니다.
3. **`[2]` 레거시 프록시 정리 (Unlink Legacy Settings)**
   * (참고) 구버전(프록시 기반)을 사용하셨던 분들만 기존 찌꺼기 주소를 청소하기 위해 한 번 실행해 주시면 됩니다. 신규 사용자는 누를 필요가 없습니다.

## 💎 주요 핵심 기술: Zero-Interference 아키텍처

* **안정성 100% (No Error 10061)**
  * 과거 버전과 달리 API 요청을 중간에 가로채지 않습니다. Codex가 자체적으로 기록하는 `state_5.sqlite` 데이터베이스를 **순수 읽기 전용(Read-Only)**으로 모니터링하여, 프록시 충돌이나 네트워크 연결 거부(`os error 10061`)를 원천 차단했습니다.
* **실시간 자동 새로고침 (SSE 연동)**
  * SSE(Server-Sent Events) 스트리밍을 통해 브라우저 새로고침 없이 실시간으로 토큰 통계와 세션 진행 상황을 업데이트합니다.
* **비용 시각화**
  * 일일 토큰 소비량, 모델별 사용 효율 분석 등 시각화된 차트를 통해 내 코딩 프로젝트에서 얼마나 많은 비용이 소요되고 있는지 직관적으로 보여줍니다.

## 📂 데이터 저장소 및 경로

Token Tracker는 이제 자체 DB를 생성하지 않고, **Codex의 기본 DB**를 읽어옵니다.

* **타겟 데이터 소스**: `~/.codex/state_5.sqlite` (윈도우 기준: `C:\Users\<User>\.codex\state_5.sqlite`)
* 개인 데이터 수집이나 외부 전송은 일절 없으며 100% 오프라인(로컬)으로 렌더링됩니다.

## 💡 자주 묻는 질문 (FAQ)

> [!TIP]
> 아키텍처 리팩토링(직접 DB 조회 방식)으로 인해 사용성이 압도적으로 간편해졌습니다.

### Q1. 예전처럼 로그인(`token login`)과 IDE 연동(`token link`)을 해야 하나요?
**A: 아니요! 전부 필요 없어졌습니다.**
Token Tracker는 더 이상 프록시 서버 형태로 중간에 개입하지 않습니다. 대신 Codex가 자체적으로 보관하는 `state_5.sqlite` 데이터를 직접 읽어오므로, 복잡한 API 키 인증이나 프록시 설정이 원천적으로 불필요합니다.

### Q2. 대시보드 웹 브라우저 창을 닫아두어도 사용량이 누적되나요?
**A: 네, 완벽하게 누적 저장됩니다.**
Token Tracker는 단순히 Codex의 DB를 '시각화하여 보여주는' 역할만 수행합니다. 대시보드를 꺼두어도 Codex가 알아서 토큰을 계속 기록하므로 데이터 유실이 없습니다. 나중에 대시보드를 다시 켜면 최신 통계가 한 번에 반영됩니다.

### Q3. "대상 컴퓨터에서 연결을 거부했습니다 (Error 10061)" 에러가 왜 사라졌나요?
**A: 통신 가로채기를 중단했기 때문입니다.**
기존에는 Token Tracker 프록시 서버가 꺼져있을 때 IDE가 프록시로 통신을 시도하다 오류를 뿜었습니다. 이제는 IDE가 Codex와 다이렉트로 통신하며, 대시보드는 백그라운드 DB만 읽는 구조(Read-Only)이므로 통신 단절 오류가 완벽히 사라졌습니다.

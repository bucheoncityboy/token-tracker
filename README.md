# 🪨 Token Tracker (토큰 트래커)

> AI 코딩 중 소모되는 토큰량과 비용을 100% 무자각으로 실시간 감시하고 시각화하는 초현대적 토큰 트래킹 솔루션입니다.
> 
> **ChatGPT Plus 구독(Responses API) 및 일반 API Key를 모두 지원**하며, 복잡한 설정 없이 더블클릭과 전역 `token` 명령어 단 하나로 모든 개발 환경을 제어합니다.

---

## 🤖 AI Agent / LLM Quick Setup (귀찮으면 그냥 복사해서 붙여넣기 하세요)

> [!NOTE]
> AI 에이전트(LLM)나 코딩 어시스턴트가 이 저장소를 읽고 있다면, 아래 스크립트 블록을 그대로 실행하여 원클릭으로 설치 및 연동을 완료해 주세요.

### 윈도우 파워쉘(PowerShell) 원클릭 통합 설치 스크립트
프로젝트 루트 디렉토리에서 아래 파워쉘 명령어를 복사하여 터미널에 붙여넣기 하세요. 의존성 설치, PowerShell 전역 `token` 명령어 등록, 윈도우 UTF-8 한글 깨짐 방지 설정까지 한 번에 완료됩니다.

```powershell
npm install; node src/setup-profile.js; Write-Host "`n✓ Installation and global token command setup completed successfully!" -ForegroundColor Green
```

설치가 끝난 후 다른 LLM/에이전트가 즉각 호출할 수 있는 단독 명령어 모음입니다:
* **개발 환경 연동 및 해제**:
  * VS Code & Continue 자동 연동: `token link`
  * 기존 설정으로 완벽 복구: `token unlink`
* **로그인 및 서버 제어**:
  * OpenAI Plus 구독 로그인: `token login`
  * 실시간 웹 대시보드 서버 가동 및 실행: `token dashboard`
  * 로그아웃 및 백그라운드 서버 종료: `token logout`
  * 현재 로그인 세션 상태 확인: `token status`

## 🚀 3초 완성 초간단 설치법 (Windows)

프로젝트 폴더 내에 포함된 **`setup.bat`** 파일을 마우스로 **더블클릭**만 하시면 설치가 완료됩니다!
* **자동 완료되는 작업**:
  1. 필요한 npm 패키지 자동 설치 (`npm install`)
  2. PowerShell 프로필(`$PROFILE`)에 전역 **`token`** 단축 명령어 자동 등록 및 한글 인코딩(UTF-8) 깨짐 완벽 방지 세팅

## ⚡ 핵심 사용법 & 표준 워크플로우

파워쉘 터미널을 열고 **`token`**을 입력하면 모든 작업이 가능한 인터랙티브 메뉴가 열립니다.

```powershell
token
```

### 🔄 표준 워크플로우 (이 순서대로 누르면 끝!)

```
[1] 로그인 (Login) ──▶ [3] 자동 연동 (Link) ──▶ [2] 대시보드 가동 ──▶ 💻 자유로운 코딩 ──▶ [4] 연동 해제 (Unlink)
```

1. **`[1]` 로그인 (OpenAI Subscription Login)**
   * ChatGPT Plus 구독자라면 브라우저 자동 인증을 통해 **무제한 구독 모드**로 연동되며, 일반 API 키 등록 방식도 완벽 지원합니다.
2. **`[3]` 자동 연동 (Auto-Link IDE Settings)**
   * 귀찮게 설정을 바꿀 필요 없이 클릭 한 번으로 VS Code와 Continue의 OpenAI API Base URL을 로컬 프록시 주소(`http://localhost:3000/v1`)로 자동 연동합니다.
3. **`[2]` 대시보드 가동 (Open Real-time Dashboard)**
   * 실시간 다크 테마 글래스모피즘 웹 대시보드 서버가 백그라운드에 구동되고 브라우저 창이 열립니다.
4. **`💻` 코딩 및 실시간 트래킹**
   * VS Code나 IDE에서 평소처럼 Codex 또는 AI 어시스턴트로 코딩하시면 소모되는 토큰이 실시간으로 집계됩니다.
5. **`[4]` 연동 해제 (Auto-Unlink IDE Settings)**
   * 코딩이 끝난 후, 원래 사용하던 본래의 개발 환경 설정 상태로 깔끔하게 원상 복구(백업 원복)합니다.

## 💎 주요 핵심 기술

* **OpenAI API 투명 어댑터 프록시 (Auto-tracking)**
  * 외부 AI 코딩 툴이 쏘는 일반 `/v1/chat/completions` 요청을 가로채 ChatGPT Plus **Responses API 스펙**으로 실시간 지능형 중계 및 역변환을 수행합니다.
* **지능형 스트리밍 에뮬레이션 (Streaming Emulation)**
  * 구독제 API가 지원하지 않는 실시간 스트리밍(`stream: true`)을 위해, 비-스트리밍 응답을 **10~25ms 단위의 동적 타이핑 SSE 청크**로 쪼개어 실시간으로 에뮬레이션 전송해 줍니다. IDE의 타이핑 모션이 100% 보존됩니다.
* **실시간 SSE 다크 테마 대시보드**
  * SSE(Server-Sent Events) 프로토콜을 사용해 새로고침 없이 실시간으로 토큰 통계, 세션 흐름, Q&A 로그를 초현대적인 UI로 볼 수 있습니다.

## 📂 파일 및 데이터 저장소

모든 토큰 데이터는 로컬 환경에 안전하게 SQLite3로 보관됩니다.

* **설정 및 데이터 디렉토리**: `~/.token-tracker/` (C:\Users\<User>\.token-tracker)
  * `tokens.db`: SQLite 데이터베이스 (모든 토큰 및 질문 로그)
  * `config.json`: 로컬 설정 및 API 구성
  * `oauth-token.json`: Plus 구독 인증 정보

## 💡 자주 묻는 질문 (FAQ / Q&A)

> [!TIP]
> Token Tracker를 사용하면서 겪을 수 있는 환경 설정 및 작동 원리에 대한 궁금증을 한눈에 해결해 드립니다.

### Q1. 최초 1회 자동 연동(`[3] Link`)을 완료한 뒤, 매번 코딩을 시작할 때마다 다시 연동해야 하나요?
* **A**: **아닙니다! 최초 1회만 하시면 됩니다.**
  * `[3]`번을 눌러 연동을 완료하면 VS Code 설정 파일(`settings.json`)의 주소 자체가 `http://localhost:3000/v1` 로컬 프록시 주소로 안전하게 고정됩니다.
  * 따라서 매번 연동을 반복할 필요가 없으며, 다음날부터는 파워쉘에서 **`token` ➔ `[2] 대시보드 가동`**만 실행하면 번거로운 과정 없이 바로 코딩과 실시간 트래킹을 이어서 진행하실 수 있습니다.

### Q2. ChatGPT Plus 인증이 만료되어 재로그인(`[1] Login`)을 수행한 경우, IDE 연동도 다시 진행해야 하나요?
* **A**: **아닙니다! 다시 연동할 필요가 전혀 없습니다.**
  * VS Code가 바라보는 **프록시 연동 상태**와 로컬에 저장되는 **로그인 인증 정보**는 완벽하게 격리되어 관리됩니다.
  * 재로그인을 하여 토큰이 갱신되더라도 프록시 주소 자체는 변하지 않으므로, 로컬 서버가 알아서 갱신된 새로운 인증 정보로 실시간 중계 처리를 이어갑니다. 

### Q3. 백그라운드 서버를 꼭 실행 중이어야만 하나요?
* **A**: **네, 필수입니다.**
  * 자동 연동(`[3] Link`) 상태에서는 IDE의 모든 API 호출 트래픽이 로컬 백그라운드 프록시 서버(`localhost:3000`)를 관통하여 OpenAI로 릴레이됩니다.
  * 따라서 백그라운드 서버가 종료되어 있으면 IDE에서 네트워크 통신 오류가 발생하므로, 코딩을 하실 때는 항상 **`[2] 대시보드 가동`**을 실행해 둔 상태여야 합니다.

### Q4. 대시보드 웹 브라우저 창을 닫아두어도 사용량이 누적되어 업데이트되나요?
* **A**: **네, 완벽하게 누적 저장됩니다.**
  * 대시보드 브라우저 탭을 꺼두었더라도 백그라운드 서버가 계속 실행 중이라면, IDE에서 발생하는 모든 대화 로그와 소모 토큰 데이터는 로컬 SQLite 데이터베이스(`tokens.db`)에 누락 없이 안전하게 상시 기록됩니다.
  * 코딩을 마친 후 혹은 나중에 다시 `[2]`번을 눌러 대시보드를 열면, 닫혀있던 동안 누적되었던 최신 통계와 내역들이 한 번에 온전히 복원되어 나타납니다.

### Q5. 대시보드 브라우저 창을 띄워놓고 코딩하면 실시간으로 반영되나요?
* **A**: **네, 100% 실시간으로 새로고침 없이 즉각 반영됩니다.**
  * SSE(Server-Sent Events) 이벤트 감지 기술이 탑재되어 있어, 브라우저 화면을 켜놓고 코딩하면 API 응답이 도달하는 즉시 새로고침을 누르지 않아도 사용량 통계 카드, 누적 차트, 상세 대화 로그 목록이 물 흐르듯 실시간으로 갱신됩니다.

### Q6. 코딩 세션을 모두 끝마치고 원래 기존 세팅으로 되돌리려면 어떻게 하나요?
* **A**: 작업 종료 시 아래 순서대로 메뉴를 호출하시면 가장 안전하고 깔끔합니다:
  1. **`[4] Auto-Unlink` 선택** ➔ 로컬 프록시 주소를 원래 쓰시던 깔끔한 순정 설정 상태로 원상 복구합니다.
  2. **`[5] Logout & Close Background Server` 선택** ➔ 백그라운드에서 가동 중이던 프록시 포트와 노드 서버 프로세스를 완전히 종료하고 메모리를 정리합니다.

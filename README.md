# 🪨 Token Tracker (토큰 트래커)

> AI 코딩 중 소모되는 토큰량과 비용을 100% 무자각으로 실시간 감시하고 시각화하는 초현대적 토큰 트래킹 솔루션입니다.
> 
> **Codex DB를 직접 읽어오는 초경량 아키텍처**로 재탄생하여, 복잡한 프록시 설정이나 오류 없이 더블클릭과 전역 `token` 명령어 단 하나로 모든 개발 환경을 제어합니다.

---

## 🤖 AI Agent / LLM Quick Setup (귀찮으면 그냥 복사해서 붙여넣기 하세요)

> [!NOTE]
> AI 에이전트(LLM)나 코딩 어시스턴트가 이 저장소를 읽고 있다면, 아래 스크립트 블록을 그대로 실행하여 원클릭으로 설치 및 연동을 완료해 주세요.

### 윈도우 파워쉘(PowerShell) 원클릭 통합 설치 스크립트
프로젝트 루트 디렉토리에서 아래 파워쉘 명령어를 복사하여 터미널에 붙여넣기 하세요. 의존성 설치, PowerShell 전역 `token` 명령어 등록, 윈도우 UTF-8 한글 깨짐 방지 설정까지 한 번에 완료됩니다.

```powershell
npm install; node src/setup-profile.js; Write-Host "`n✓ Installation and global token command setup completed successfully!" -ForegroundColor Green
```

설치가 끝난 후 다른 LLM/에이전트가 즉각 호출할 수 있는 명령어입니다:
* **대시보드 실행**: `token dashboard`
* **통계 및 상태 확인**: `token status`, `token ls`
* **기존 설정 정리 (선택)**: `token unlink` (과거 프록시 기반 버전을 사용했던 경우 찌꺼기 설정 정리)

## 🚀 3초 완성 초간단 설치법 (Windows)

프로젝트 폴더 내에 포함된 **`setup.bat`** 파일을 마우스로 **더블클릭**만 하시면 설치가 완료됩니다!
* **자동 완료되는 작업**:
  1. 필요한 npm 패키지 자동 설치 (`npm install`)
  2. PowerShell 프로필(`$PROFILE`)에 전역 **`token`** 단축 명령어 자동 등록 및 한글 인코딩(UTF-8) 깨짐 완벽 방지 세팅

## ⚡ 핵심 사용법 & 표준 워크플로우

파워쉘 터미널을 열고 **`token`**을 입력하면 대시보드 메뉴가 열립니다.

```powershell
token
```

### 🔄 표준 워크플로우

```
[1] 대시보드 가동 ──▶ 💻 자유로운 코딩 ──▶ 📊 실시간 토큰 확인
```

1. **`[1]` 대시보드 가동 (Open Real-time Dashboard)**
   * 실시간 다크 테마 글래스모피즘 웹 대시보드 서버가 백그라운드에 구동되고 브라우저 창이 열립니다.
2. **`💻` 코딩 및 실시간 트래킹**
   * VS Code나 IDE에서 평소처럼 Codex 또는 AI 어시스턴트로 코딩하세요. Token Tracker가 백그라운드에서 조용히 소모되는 토큰을 모니터링합니다.
3. **`[2]` 연동 해제 (Auto-Unlink IDE Settings)**
   * (참고) 과거 버전에서 프록시 기반으로 사용하셨던 분들만 기존 로컬 프록시 주소를 정리하기 위해 한 번 실행하시면 됩니다. 이제는 더 이상 프록시를 통하지 않고 Codex 자체 DB를 읽어옵니다.

## 💎 주요 핵심 기술

* **Zero-Interference 아키텍처 (안정성 100%)**
  * 이전 버전과 달리, API 요청을 중간에 가로채지 않습니다. Codex가 자체적으로 저장하는 `state_5.sqlite` 데이터베이스를 **읽기 전용(Read-Only)**으로 모니터링하여 오류 발생 가능성을 원천 차단했습니다 (`os error 10061` 완벽 해결).
* **자동 새로고침 (SSE 연동)**
  * SSE(Server-Sent Events) 프로토콜을 사용해 새로고침 없이 실시간으로 토큰 통계 및 세션 흐름을 대시보드에 즉각 반영합니다.
* **초현대적인 실시간 다크 테마 대시보드**
  * 시각화된 차트와 세션 리스트를 통해 프로젝트별/모델별 누적 토큰 소모량과 추정 비용을 즉시 확인할 수 있습니다.

## 💡 자주 묻는 질문 (FAQ / Q&A)

> [!TIP]
> 새로운 아키텍처로 인해 기존 버전보다 훨씬 더 사용하기 간편해졌습니다.

### Q1. 예전처럼 로그인(`token login`)과 IDE 연동(`token link`)을 해야 하나요?
* **A**: **아니요! 전부 필요 없어졌습니다.**
  * Token Tracker는 더 이상 프록시 형태로 중간에 개입하지 않습니다. 대신 Codex가 자체적으로 보관하는 `state_5.sqlite`를 직접 조회하므로, 복잡한 인증 설정이나 IDE 연동이 원천적으로 불필요해졌습니다.

### Q2. 대시보드 웹 브라우저 창을 닫아두어도 사용량이 누적되나요?
* **A**: **네, 완벽하게 누적 저장됩니다.**
  * Token Tracker는 단순히 Codex의 DB를 '보여주는' 역할만 합니다. 대시보드를 닫아두어도 Codex 자체가 토큰을 계속 기록하므로 아무런 데이터 유실이 없습니다. 나중에 대시보드를 다시 켜면 모든 통계가 정상 표시됩니다.

### Q3. 에러 10061 (연결 거부)가 자꾸 떴었는데 어떻게 된 건가요?
* **A**: **영구적으로 해결되었습니다.**
  * 기존에는 Token Tracker 프록시 서버가 꺼져있을 때 IDE가 프록시로 통신을 시도하다 오류를 뿜었습니다. 이제는 프록시 구조 자체를 삭제하고 다이렉트 통신을 사용하므로, 해당 에러는 더 이상 발생하지 않습니다.
